enum struct ogcwrap::asnd::ASNDReturnValue : s8
strong

Enum to specify common return values of ASND functions.

Enumerator
OK 

This return value is used to indicate that a function was successful.

Invalid 

This return value is used to indicate that an input to a function is invalid.

NotAVoice 

This return value is used to indicate that the voice number given is invalid.

VoiceBusy 

This return value is used to indicate that a voice is busy.

VoiceUnused 

This return value is used to indicate that a voice is currently unused and is ready to be used.

VoiceWorking 

This return value is used to indicate that a voice is working on playing samples.

VoiceWaiting 

This return value is used to indicate that a voice is waiting for (more samples?).

◆ ASNDVoiceFormat

enum struct ogcwrap::asnd::ASNDVoiceFormat : u8
strong

Enum to specify the format of PCM samples that the DSP will use.

Enumerator
MonoU8 

Specifies that the PCM samples only have one sound channel (mono) and that they have unsigned 8-bit values.

MonoS8 

Specifies that the PCM samples are in mono and that they have signed 8-bit values.

MonoBE16 

Specifies that the PCM samples are in mono and that they have unsigned big-endian 16-bit values.

MonoLE16 

Specifies that the PCM samples are in mono and that they have unsigned little-endian 16-bit values.

StereoU8 

Specifies that the PCM samples have two sound channels (stereo) and that they have unsigned 8-bit values.

StereoS8 

Specifies that the PCM samples are in stereo and that they have signed 8-bit values.

StereoBE16 

Specifies that the PCM samples are in stereo and that they have unsigned big-endian 16-bit values.

StereoLE16 

Specifies that the PCM samples are in stereo and that they have unsigned little-endian 16-bit values.

ASNDReturnValue ogcwrap::asnd::setVoice ( u8  voice,
ASNDVoiceFormat  format,
u32  pitch,
u32  delay,
void *  samples,
u32  samplesize,
u8  left,
u8  right,
ASNDVoiceCallback  cb 
)

Sets up a voice with the given arguments and starts playing after the time specified in delay has passed.

Note
All voices play audio in 16-bit stereo, regardless of the format they receive.
Periverbatim from libASND: "<i>callback<i> is used to implement a double buffer. When the second buffer is empty, the callback function is called with the voice number as an argument. You can use <tt>void <i>callback</i> (s32 voice)</tt> to call ASND_AddVoice() and add one voice to the second buffer. When the callback is non-NULL, the voice never stops and returns SND_WAITING if successful on timeout condition."
Parameters
[in]voiceVoice slot to use.
[in]formatFormat of the samples in samples, as specified in ASNDVoiceFormat.
[in]pitchPitch to use for the voice, in Hz.
[in]delayAmount of milliseconds to wait before the voice should start playing.
[in]samplesBuffer of sound samples. Must be aligned and padded to 32 bytes.
[in]samplesizeThe size of the samples buffer, in bytes.
[in]leftVolume of the left stereo channel.
[in]rightVolume of the right stereo channel.
[in]cbCallback for (that double buffer thing?)
Returns
::Invalid if either of the following is true:
voice is not a valid voice slot
samplesize is less than 1 (0 or negative)

::OK otherwise.
